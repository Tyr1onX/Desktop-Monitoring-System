# Roadmap

## 当前阶段

项目当前处于 Pre-research / System design 阶段。目标是先明确系统边界、技术路线和实验计划，再逐步补充代码与验证材料。

## Phase 1: 仓库整理与方案文档

- 明确 README 中的项目状态。
- 整理项目简介、背景、架构和模块设计。
- 补充方案摘要、架构说明和后续计划文档。
- 避免将项目描述成已经完成的系统。

## Phase 2: 硬件采集原型

- 选择 ESP32 / ESP32-S3 开发板。
- 验证 Wi-Fi CSI 采集能力。
- 设计 CSI 数据字段和消息格式。
- 记录开发环境、依赖版本和硬件连接方式。

## Phase 3: MQTT 通信链路

- 设计 topic 命名规范。
- 选择本地或云端 MQTT broker。
- 实现 ESP32 数据发布原型。
- 实现 Python 数据订阅原型。
- 记录异常处理和重连策略。

## Phase 4: Python 数据处理与可视化

- 实现 CSI 数据解析脚本。
- 保存采集数据为 CSV、JSONL 或其他实验格式。
- 实现基础实时曲线展示。
- 增加离线回放能力。
- 记录不同坐姿和环境条件下的观察结果。

## Phase 5: 数据集与标注

- 定义坐姿类别。
- 设计采集流程和标注规范。
- 控制采集距离、设备位置和时间窗口。
- 记录环境变量和实验条件。
- 建立可复现实验数据集。

## Phase 6: CNN-LSTM 预研

- 设计输入特征和时间窗口。
- 实现基础训练脚本。
- 划分训练集、验证集和测试集。
- 记录实验条件和结果。
- 只有在经过验证后，才在 README 中说明具体指标。

## Phase 7: 系统集成

- 串联硬件采集、MQTT、Python 后端和可视化。
- 增加配置文件和运行说明。
- 增加错误处理和日志。
- 补充演示数据或最小可运行示例。

## Phase 8: 验证与公开说明

- 明确测试环境。
- 补充实验报告。
- 说明系统限制和适用范围。
- 避免夸大部署状态、识别效果或稳定性。
