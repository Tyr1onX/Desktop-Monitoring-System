# Desktop-Monitoring-System Proposal Summary

## 项目定位

Desktop-Monitoring-System 当前定位为方案设计与技术预研项目，目标是探索基于 ESP32 和 Wi-Fi CSI 的非接触式桌面坐姿监测方案。

当前仓库暂未形成完整可运行系统，主要用于整理项目背景、技术路线、架构设计和后续实验计划。

## 问题背景

在学习和办公场景中，用户可能长期保持不良坐姿。摄像头方案可能涉及隐私问题，穿戴式设备可能影响体验，压力传感方案可能依赖特定座椅或硬件安装。

Wi-Fi CSI 提供了一种可预研的非接触式感知方向。人体姿态变化可能影响无线信道状态，因此可以尝试通过 CSI 信号变化分析坐姿状态。

## 方案目标

- 预研 ESP32 / ESP32-S3 采集 Wi-Fi CSI 数据的可行性。
- 设计硬件采集端、MQTT 通信链路、Python 后端和可视化模块。
- 探索基于 CSI 时序数据的坐姿识别方法。
- 为后续数据采集、模型训练和系统实现提供设计依据。

## 技术路线

1. 使用 ESP32 / ESP32-S3 作为 CSI 数据采集端。
2. 将采集到的 CSI 数据封装为结构化消息。
3. 通过 MQTT 将数据发送到后端或上位机。
4. 使用 Python 接收、解析、记录和预处理数据。
5. 使用可视化工具观察 CSI 波形变化。
6. 预研 CNN-LSTM 等模型用于时序姿态分类。

## 预期模块

- 硬件采集模块
- MQTT 通信模块
- Python 数据处理模块
- 实时或离线可视化模块
- 数据集构建与标注流程
- CNN-LSTM 姿态识别实验模块

## 当前阶段

当前阶段侧重方案设计和技术预研。仓库中的公开内容不代表系统已经完成部署，也不代表已经具备稳定识别能力。

## 风险与限制

- CSI 信号对环境变化敏感。
- 姿态差异与信号变化之间的关系需要实验验证。
- 不同设备、位置和房间环境可能导致模型泛化困难。
- 需要规范的数据采集与标注流程。
- 需要避免在未验证前声明准确率或稳定性。
