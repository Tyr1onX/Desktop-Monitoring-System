# System Architecture

## 架构状态

本文档描述 Desktop-Monitoring-System 的预研架构。当前内容是设计方案，不表示仓库已经包含完整实现。

## 总体架构

```txt
+-------------------+       +-------------+       +--------------------+
| ESP32 / ESP32-S3  | ----> | MQTT Broker | ----> | Python Backend     |
| CSI Collection    |       | Message Bus |       | Parse / Store Data |
+-------------------+       +-------------+       +--------------------+
                                                        |
                                                        v
                                             +----------------------+
                                             | Visualization        |
                                             | Signal Observation   |
                                             +----------------------+
                                                        |
                                                        v
                                             +----------------------+
                                             | CNN-LSTM Research    |
                                             | Posture Recognition  |
                                             +----------------------+
```

## ESP32 / ESP32-S3

ESP32 端计划承担 CSI 数据采集职责：

- 开启 Wi-Fi CSI 相关能力。
- 采集子载波级别的信号信息。
- 对原始数据进行初步封装。
- 将数据通过 MQTT 发布到指定 topic。

需要进一步验证的内容：

- 不同 ESP32 开发框架下 CSI API 的可用性。
- CSI 数据格式、采样频率和稳定性。
- 设备摆放位置对信号质量的影响。

## MQTT

MQTT 计划作为硬件与后端之间的轻量级通信通道。

预研点包括：

- topic 命名规范，例如 `desktop-monitoring/csi/raw`。
- QoS 等级选择。
- 消息格式设计。
- 断线重连与异常处理。
- 公共 broker 与本地 broker 的差异。

## Python 后端

Python 后端设计为数据接收与处理中心：

- 订阅 MQTT topic。
- 解析 CSI 消息。
- 做基础数据校验。
- 将数据保存为后续实验可用的格式。
- 为可视化和模型实验提供数据入口。

计划使用或预研的库：

- `paho-mqtt`
- `numpy`
- `matplotlib`
- `scipy` 或其他信号处理工具

## 可视化

可视化模块用于辅助观察数据质量和姿态差异。

计划能力：

- 展示 CSI 子载波幅值变化。
- 支持实时曲线观察或离线数据回放。
- 帮助判断采样是否正常、噪声是否明显、动作是否可区分。

## 信号预处理

CSI 原始数据可能包含噪声和环境干扰，后续计划预研：

- 缺失值处理
- 异常值过滤
- Butterworth filter
- 滑动窗口切片
- 归一化
- 特征构造

## CNN-LSTM

CNN-LSTM 是计划预研的姿态识别模型方向：

- CNN 用于提取局部信号模式。
- LSTM 用于建模姿态变化中的时间序列信息。
- 输入可以设计为按时间窗口组织的 CSI 特征矩阵。
- 输出可以设计为坐姿类别或异常坐姿提示。

当前仓库尚未提供完整模型训练、验证或部署代码。

## 数据流

1. ESP32 采集 CSI 数据。
2. ESP32 将数据封装并发布到 MQTT broker。
3. Python 后端订阅 topic 并解析消息。
4. 数据被缓存、保存或送入可视化模块。
5. 清洗后的数据用于后续 CNN-LSTM 实验。

## 待验证问题

- CSI 是否能在目标桌面场景中稳定区分坐姿。
- 采样频率是否足够支持时序识别。
- MQTT 链路是否满足实验数据传输要求。
- 模型在不同用户、位置和环境下是否具备泛化能力。
